otter-examples walkthrough

Two apps, one UI: layer-shell overlay and XDG window. Snippets below say what each file does. More background in the official developer guide.

New here? Start with the step-by-step tutorial, then come back for file-by-file detail.

What you are building

Otter Shell apps talk to Wayland, describe UI with Surface Description (SD) trees, and paint through shared libraries instead of hand-writing draw commands. The example shows:

  • A rounded panel, Otter logo, counter label, and increment button with hover/press styling.
  • Pointer cursor changes over the button; partial damage on hover and click.
  • Two shell backends: layer overlay and xdg window.

The loop is always: connect, create a surface, build an SD tree on configure, rasterize dirty rects, commit. Shell code differs; UI code is shared.

Step 3

Wire the build: build.zig

const std = @import("std");

pub fn build(b: *std.Build) void {
    const target = b.standardTargetOptions(.{});
    const optimize = b.standardOptimizeOption(.{});
    const c_allocator = b.option(enum { libc, jemalloc, mimalloc }, "c_allocator", "...") orelse .libc;

    const utils = b.dependency("otter_utils", .{ .target = target, .optimize = optimize });
    const ui = b.dependency("otter_ui", .{
        .target = target,
        .optimize = optimize,
        .enable_dbus = false,
        .enable_pipewire = false,
        .enable_pam = false,
        .c_allocator = c_allocator,
    });
    // ... same pattern for geo, render, theme, wayland ...

    const imports = [_]std.Build.Module.Import{
        .{ .name = "otter_utils", .module = utils.module("otter_utils") },
        // one entry per dependency ...
    };

    const root_module = b.createModule(.{
        .root_source_file = b.path("src/main.zig"),
        .target = target,
        .optimize = optimize,
        .imports = &imports,
    });

    const exe = b.addExecutable(.{ .name = "otter-examples", .root_module = root_module });
    b.installArtifact(exe);
}
standardTargetOptions / standardOptimizeOption
Respect -Dtarget= and -Doptimize= from the CLI. Use ReleaseFast for Otter; debug builds can segfault on big transitive compiles.
c_allocator option
Lets you point C libraries (FreeType, etc.) at jemalloc/mimalloc if you are profiling heap churn.
b.dependency("otter_ui", ... enable_dbus = false ...)
Pulls the UI stack without D-Bus/PipeWire/PAM. Faster builds for a minimal surface-only app.
imports array
Maps Zig @import("otter_ui") names to modules. Name must match what source files use.
No linkSystemLibrary in the app
otter_wayland already links wayland-client and xkbcommon on its module. Zig propagates that to the exe; don't repeat it in your build.zig.
Step 4

Entry point: src/main.zig

pub fn main(init: std.process.Init) !void {
    utils.io.install(init.io);

    const args = try init.minimal.args.toSlice(init.arena.allocator());
    const mode = if (args.len > 1) Mode.parse(args[1]) orelse {
        usage();
        return error.UnknownMode;
    } else blk: {
        usage();
        break :blk .window;
    };

    switch (mode) {
        .layer => try layer_app.run(init.gpa),
        .window => {
            var xdg_opts: xdg_app.Options = .{};
            for (args[2..]) |arg| {
                if (ui.debugOverlayModeFromArg(arg)) |overlay_mode| {
                    xdg_opts.debug_overlay_mode = overlay_mode;
                }
            }
            try xdg_app.run(init.gpa, xdg_opts);
        },
    }
}
std.process.Init
Zig 0.16 passes I/O, GPA, and args through std.process.Init. No global std.os.argv.
utils.io.install(init.io)
Register process I/O before any Otter file or network call.
init.minimal.args.toSlice
Args slice from the init arena; freed on exit.
Mode.parse
Accepts layer, window, or xdg. Same binary, two demos.
init.gpa
General purpose allocator for the long-lived shell app. Short one-shot tools sometimes use an arena instead.
layer_app.run / xdg_app.run
Each shell owns its own Wayland objects; UI stays shared underneath.

pub fn onPointerPress(self: *Demo, ui_state: *UiState, point: Point, damage: *DamageTracker) PressResult {
    const press = ui_state.dispatch(.{ .button_press = .{ .point = point, .button = 1 } });
    if (press.id.eql(Ids.increment)) {
        self.damageIncrement(damage); // pressed fill via input.active
        self.counter +%= 1;
        self.damageCounter(damage);
        return .handled;
    }
    return .none;
}

pub fn onPointerRelease(self: *Demo, ui_state: *UiState, point: Point, damage: *DamageTracker) bool {
    const was_active = ui_state.input.active.eql(Ids.increment);
    _ = ui_state.dispatch(.{ .button_release = .{ .point = point, .button = 1 } });
    if (was_active) {
        self.damageIncrement(damage);
        return true;
    }
    return false;
}

Step 6

Draw pipeline: src/ui/draw.zig

pub fn draw(frame: Frame) void {
    if (!frame.damage.hasDamage() and !frame.damage.isFullDamage()) return;

    var acquired = frame.renderer.acquire() orelse return;
    defer acquired.release();

    const effective = frame.damage.getEffectiveDamage();
    var ui_frame = frame.ui_state.begin(.{ .viewport = viewport, .font = frame.font, ... });

    if (effective.full) ui_frame.clear(background);

    const root = frame.demo.buildCard(frame.ui_state, frame.shell_label);
    _ = ui_frame.render(&root, card_rect) catch {};
    ui_frame.finish() catch {};

    if (!effective.full)
        for (damage_rects) |rect| acquired.surface.fillRectLogical(rect, background);

    frame.ui_state.rasterize(&acquired.surface, damage_rects, effective.full);
    frame.renderer.submit(frame.surface, frame.damage);
    frame.damage.commitFrame();
    frame.demo.captureRects(frame.ui_state);
}
Early return when no damage
Guards against accidental draw calls. Combined with frame callbacks, the app stays idle when nothing changed.
renderer.acquire / release
Double-buffered SHM pool. acquire maps the back buffer for this frame.
getEffectiveDamage
Unions current + previous frame rects for double buffering. Compositor may still show the old buffer briefly.
ui_state.begin, render, finish
SD emit phase: builds draw commands in UiState, records element rects and hit regions.
fillRectLogical on partial damage
Erases old pixels in dirty rects before rasterize. Layer uses transparent fill; XDG uses theme background.
ui_state.rasterize(..., damage_rects, full)
Plays commands back with scissor culling to damaged areas when full == false.
commitFrame
Rotates damage lists for next frame. Call once per successful submit.
Step 7

Frame coalescing: src/shell/frame.zig

pub fn request(self: *Driver) void {
    self.needs_redraw = true;
    if (self.frame_callback != null) return;
    self.frame_callback = surface.frame() catch return;
    self.frame_callback.?.setListener(*Driver, frameListener, self);
    surface.commit();
}
needs_redraw flag
Input sets dirty state; paint happens on the frame callback (except first configure; see drawNow).
wl_surface.frame
Ask compositor to call back at display refresh. Batches rapid hover events into one paint.
frame_callback != null guard
Only one pending frame request. Standard Wayland pattern.
drawNow
Synchronous paint before the event loop runs. Configure handlers need visible pixels immediately.

Layer-shell app: src/shell/layer_app.zig

Layer surfaces are overlays: notifications, popups, bar-adjacent panels. They are anchored, not moved by the user.

const layer = try layer_shell.getLayerSurface(surface, null, .top, "otter-examples-layer");
layer.setSize(self.width, self.height);
layer.setAnchor(.{ .top = true, .right = true });
layer.setMargin(48, 32, 0, 0);
layer.setKeyboardInteractivity(.none);
try ow.setSurfaceTransparent(compositor, surface);
getLayerSurface(..., .top, namespace)
Creates a layer-shell surface on the top layer. Namespace helps compositors identify your overlay.
setAnchor top + right
Pins the surface to the top-right of the output. Margins offset from the screen edge in logical px.
setKeyboardInteractivity(.none)
No keyboard focus. Fine for a click-only demo.
setSurfaceTransparent
Allows rounded panel corners to show desktop through alpha. Pair with transparent background in draw.zig.
layerListener configure
On configure: ack serial, resize renderer, markFullDamage, drawNow.
onPointerButton outside button
demo.onPointerPress returns .none and the layer quits. Press and release both forward into demo.zig; shell calls applyPointerCursor() on motion.
poll(-1) event loop
Blocks until Wayland has work. No idle timer.

Needs WAYLAND_DISPLAY and a compositor. Opens the XDG demo. Add --inspect or --metrics for debug overlays.

Needs compositor with wlr-layer-shell (Hyprland, Sway, etc.). Top-right overlay.

Input, damage, draw

  1. Pointer moves: demo.onPointerMotion updates hover, may damage the button; shell sets cursor shape
  2. Button press: onPointerPress sets input.active, damages button + counter
  3. Button release: onPointerRelease clears active, damages button again
  4. Frame callback runs draw.zig, partial rasterize, captureRects
  5. Otherwise poll blocks and the app idles

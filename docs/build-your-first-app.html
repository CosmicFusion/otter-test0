Build your first Otter app

Run the template, poke the UI, add a button. Layer shell vs XDG is easier to see once you've clicked around. For what each file does, see the code walkthrough.

Two guides: this page is the tutorial; creating-wayland-apps.html is the reference.

Before you start

  • Zig 0.16.0 (zig version)
  • Wayland session with WAYLAND_DISPLAY set (Hyprland, Sway, etc.)
  • otter-examples on disk (clone or copy from the monorepo)
1

Get the template

Open a terminal:

cd otter-examples

From the monorepo that's usually:

cd /path/to/otter-shell/otter-examples

Check: build.zig, src/, and docs/ are there.

2

Build and run unit tests

Use ReleaseFast for Otter UI apps. Debug builds of the full stack often fail during compile.

zig build -Doptimize=ReleaseFast test

Exit code 0 is enough. No compositor needed for this step.

Check: command finishes without errors.

3

Run the desktop window

Normal resizable XDG window:

zig build -Doptimize=ReleaseFast run -- window

Try:

  • Hover Increment (cursor becomes a pointer; button highlight)
  • Click and hold (button shows pressed state)
  • Release (counter updates; only button + counter repaint)
  • Resize the window (card stays centered)

Check: themed card, pointer cursor on the button, visible press feedback.

4

Run the layer-shell overlay

Quit the window, then:

zig build -Doptimize=ReleaseFast run -- layer
  • Same card, pinned top-right
  • Click outside the card to dismiss

Check: floating overlay, not a normal taskbar window.

5

Add a second button (one file)

All demo UI is in src/ui/demo.zig.

5a: Add a Surface ID

In pub const Ids:

pub const reset = ui.SurfaceId.namedComptime("demo.reset");

If you add hit targets, bump .hit_regions in UiState (e.g. 8 to 9).

5b: Add the button node

In buildCard(), add a leaf (widen content if needed):

self.content[4] = .{
    .id = Ids.reset,
    .kind = .leaf,
    .layout = .{ .width = .fit, .height = .fit, .align_x = .center },
    .content = .{ .button = .{ .text = "Reset", .hovered = ui_state.input.hovered.eql(Ids.reset) } },
    .hit = .button,
};

Point the column .children slice at the new node.

5c: Handle the click

In onPointerPress(), after the increment branch:

if (press.id.eql(Ids.reset)) {
    self.counter = 0;
    self.refreshCounterText();
    self.damageCounter(damage);
    return .handled;
}

Copy the increment button's press/release damage pattern if you want hover and click styling on Reset too.

5d: Rebuild and run

zig build -Doptimize=ReleaseFast run -- window

Check: Reset sets the counter to zero. No shell file edits.

6

Try debug overlays (window mode)

Built-in SD inspector and FPS overlay:

zig build -Doptimize=ReleaseFast run -- window --inspect
zig build -Doptimize=ReleaseFast run -- window --metrics

Or Ctrl+Shift+I in the focused window: off, inspector, metrics.

  • Inspector: hover nodes for Surface IDs and rects
  • Metrics: live FPS and frame timing

Check: debug panel drawn on top of the UI.

7

Pick layer shell or XDG

BuildingUseStart from
Launcher, popup, bar widgetLayer shellshell/layer_app.zig
Settings, editor, normal windowXDG toplevelshell/xdg_app.zig

Both call the same ui/demo.zig and ui/draw.zig. Copy the shell that fits your app.

Rule of thumb: app menu + resizable = XDG. Float above everything or dock to an edge = layer shell.
